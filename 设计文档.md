## 1. 概述

软件是一台上位机 GUI，通过 SSH 登录 CK3M 控制器、进入其 `gpascii` ASCII 命令解释器，对 8 个电机做状态读取与运动控制。8 个电机按机械装置分为 4 组，每组一个标签页；其中"调焦装置"（电机 1-4）、"光栅切换装置"（电机 5）、"机械快门装置"（电机 6）三页各内嵌 3D 模型，随电机位置实时联动——调焦两套机构前后/左右移动、升降台上下移动、快门挡光片左右滑动，用于直观预览。

软件只面向单台控制器 `192.168.0.200`，单条 SSH 连接，串行下发命令。

## 3. 通信协议层（`pmac.py`）

### 3.1 连接流程（`PMAC.connect`，`pmac.py:68`）

1. `asyncssh.connect()` 建立 SSH。为兼容 CK3M 上较老的 OpenSSH，显式给出主机密钥算法集合 `_HOST_KEY_ALGS`（含 `ssh-rsa`），并用 `client_keys=()` 跳过本机公钥协商。超时统一 10 s。
2. `open_session(term_type="vt100")` 开一个伪终端，读到 shell 提示符 `:/opt/ppmac# `。
3. 发送 `gpascii -2 -f`，等待登录横幅 `STDIN Open for ASCII Input` 与一个 ACK（`\x06\r\n`）。横幅不符即抛 `ConnectionError`。

`-2 -f` 的含义：`-2` 让 gpascii 回显输入并对每条命令回两个 ACK；`-f` 行缓冲立即刷出。

### 3.2 命令收发（`exec_cmd`，`pmac.py:99`）

一条命令的完整往返：

```
写:  "<cmd>\r\n"
读:  直到出现双 ACK   "\x06\r\n\x06\r\n"
处理: 去掉命令回显行与 ACK，剩下的就是响应行
```

`asyncio.Lock`（`self._lock`）保证同一时刻只有一条命令在线，因为只有一条 gpascii 会话。响应行里若匹配到 `error #NN: ...`（正则 `_ERR`），抛 `PmacError`。

### 3.3 变量读取（`read_vars`，`pmac.py:117`）

把多个变量名用空格拼成一行下发，响应里逐行用正则 `_KV`（`name = value`）解析成 `float`。`motor_status_many`（`pmac.py:134`）据此把全 8 台的 5 个状态字段拼成**一条命令**批量读取——轮询全 8 台时这是关键优化，避免 40 次往返。

### 3.4 命令集

| 方法 | gpascii 命令 | 含义 |
| --- | --- | --- |
| `ecat_enable` | `ECAT[0].enable` | 启动 EtherCAT 总线 |
| `motor_enable(n)` | `#nj/` | 电机使能 |
| `motor_disable(n)` | `#nk` | 电机去使能 |
| `motor_move_abs(n, x)` | `#nj=x` | 绝对移动到 x |
| `motor_move_rel(n, d)` | `#nj:d` | 相对移动 d |
| 状态读 | `Motor[n].ActPos` 等 | 见 `STATUS_FIELDS` |

状态字段 `STATUS_FIELDS = (ActPos, ActVel, AmpEna, AmpFault, SoftLimit)`（`pmac.py:28`）。

> **设计决策：绝对移动用小写 `#nj=`。** v1.2 协议与硬件一一对应，确认小写 `j=` 即"绝对移动到该位置"。早期曾改用大写 `J=`，是对一次"电机冲到限位"事故的错误归因——该事故真正原因是 offset（见 §5），与大小写无关。

## 4. 设备数据模型（`motors.py`）

### 4.1 静态定义

- `MOTOR_DEFS`（`motors.py:24`）：每个电机的 `name / offset / range / needs_enable / fast`。这是默认值的唯一来源。
- `DEVICES`（`motors.py:36`）：4 个装置，每个含一组电机，列表顺序即标签页顺序；`key` 是装置标识（focus/grating/shutter/hartmann）。
- `ALL_MOTORS`：`[1..8]`，由 `DEVICES` 展开。
- `VIZ_DEVICES`（`motors.py`）：哪些装置带 3D（按装置 `key`），含 `subdir`（viewer 子目录）、`motors`（联动电机）、`kind`（`single`=升降/快门 单电机；`focus`=调焦 4 电机）。`viewer_assets(key)` / `viz_motors(key)` / `viz_kind(key)` 为其访问器。
- `DEFAULT_VIZ_BY_DEVICE`（`motors.py`）：每个 3D 装置一套默认显示参数。`single` 为 `{mm_per_unit, axis, direction}`；`focus` 为 `{mm_per_unit, dir_fb, dir_lr}`（前后/左右两方向各可翻转）。

### 4.2 配置持久化

运行时配置存于 `motor_config.json`，结构：

```json
{
  "motors": { "1": {"center": 17, "range": 6}, "...": {} },
  "viz":    {
    "grating": { "mm_per_unit": 0.0001, "axis": "x", "direction": 1 },
    "shutter": { "mm_per_unit": 0.0001, "axis": "x", "direction": 1 },
    "focus":   { "mm_per_unit": 8.0, "dir_fb": 1, "dir_lr": 1 }
  }
}
```

> `viz` 按 3D 装置 `key` 分块。`load_config` 兼容三种历史格式并自动迁移：最旧的单块 `viz`→`grating`；按电机号的 `"5"/"6"`→`grating`/`shutter`；新的按装置 key 直接合并。存盘后统一为按装置 key 的新格式。

- `load_config`（`motors.py:64`）：先取默认，再用磁盘值覆盖；字段缺失或类型不对则保留默认。文件不存在直接返回默认。
- `save_config`（`motors.py:97`）：界面改 offset / range / 显示参数后调用，回写整份。

该文件是本机运行态，不进版本库（见 `.gitignore`）。`MOTOR_DEFS` 里的默认值保证即使删掉该文件，软件行为也正确。

## 5. 坐标语义（关键）

固件命令 `#nj=X` 的 0 点对应物理 `ActPos = offset`。因此：

```
绝对输入 X（界面里填"相对中心"的值，锁死在 [-range, +range]）
下发命令          #nj=X
预期物理读数      ActPos ≈ X + offset
```

- 电机 1（offset 17）：输入 `0` → 物理 17；输入 `+6` → 物理 23；输入 `-6` → 物理 11。
- 电机 5（offset 0）：输入即物理，范围 ±2,000,000。

相对移动 `#nj:d` 的增量直接作用于物理读数，不经过 offset。

界面据此做范围校验（`MotorControl._on_abs` / `_on_rel`，`gui_motors.py:308` / `327`）：

- 绝对：输入须落在 `[-range, +range]`。
- 相对：`当前 ActPos + d` 须落在 `[offset-range, offset+range]`。

超界不下发，弹一次警告框（`MainWindow.warn`）。范围内直接下发，不再二次确认。

> **设计决策：去掉每次移动的确认弹窗。** 早期每条移动命令都弹确认，操作频繁时很烦。改为只在"超出范围"时拦截并提示，范围内静默执行。

## 6. GUI 层（`gui_motors.py`）

### 6.1 控件层级

```
MainWindow (QMainWindow)
├─ 顶栏：状态点 ● + "已连接/未连接" + [连接][断开][启动 EtherCAT]
├─ QTabWidget
│   ├─ OverviewTab  「总览」
│   ├─ DeviceTab    「调焦装置」  → 两套调焦 3D 视图 + MotorControl ×4
│   ├─ DeviceTab    「光栅切换装置」→ 升降台 3D 视图 + MotorControl ×1
│   ├─ DeviceTab    「机械快门装置」→ 快门 3D 视图 + MotorControl ×1
│   └─ DeviceTab    「哈特曼门装置」→ MotorControl ×2
└─ 底栏：☐ 保存日志到文件 + 状态文字
```

### 6.2 各类职责

**`MotorControl(QGroupBox)`（`gui_motors.py:150`）** — 单个电机的控制卡片。

- 分三区：`状态`（5 个值芯片）、`运动`（绝对/相对输入 + 按钮）、`参数`（offset / range + 保存）。
- 使能 / 去使能按钮所有电机都有（电机级 `#nj/` / `#nk`，与 EtherCAT 使能是两回事）。
- 自己做范围校验，调 `ctrl.run(...)` 下发命令。
- `update_status(st)` 被轮询调用，刷新值芯片；`AmpEna=1` 绿、`AmpFault/SoftLimit≠0` 红。
- 边界（`abs_min` 等）都是从 `cfg` 实时算的 property，改配置即时生效。

**`OverviewTab(QWidget)`（`gui_motors.py:392`）** — 总览页。

- 两个大按钮：一键使能 / 一键去使能所有电机，内部走 `run_seq` 逐台下发。
- 一张 8 行表格（`电机 / 名称 / ActPos / ActVel / AmpEna / AmpFault / SoftLimit`），`update_status(全量 dict)` 刷新，三个状态列按好/坏着色。

**`DeviceTab(QWidget)`（`gui_motors.py:489`）** — 一个装置一页。

- 电机卡片按网格排（≥3 个电机用 2 列，否则 1 列），放进可滚动区。
- 构造时按装置 `key` 查 `VIZ_DEVICES`：拿到 `assets`（QML/GLB）、`viz_motors`（联动电机）、`viz_kind`（`single`/`focus`）。有 3D 则左侧用 `QSplitter` 挂 `Viewer3DWidget` + "3D 显示"参数组，显示参数读写 `cfg["viz"][dev_key]`。
- 参数组按 `kind` 分两套：`single` 给「运动轴 + 方向」；`focus` 给「前后方向 + 左右方向」两个翻转。`push_positions(st)` 每帧把相关电机 `ActPos` 推给 viewer（`single` 调 `set_motor_position`，`focus` 对每个电机调 `set_motor_pos(motor, pos)`）。
- 卡片网格 ≥3 电机走 2 列，且每张 `setMinimumWidth(360)` 防止 `ActPos` 被挤掉。`focus` 页 4 张卡需要更宽的控制区，故其 `QSplitter` 把份额偏向控制区（3D 区相应收窄，靠相机缩放仍能容纳两套模型）。

**`MainWindow(QMainWindow)`（`gui_motors.py:612`）** — 顶层。持有 `PMAC` 实例、配置 `cfg`、轮询任务、在飞命令任务。对外提供 `run / run_seq / soft_estop / warn / log / save_cfg`，供子控件回调。

### 6.3 数据流

- **下行（命令）**：按钮点击（`@asyncSlot`）→ `MainWindow.run(label, fn, *args)` → 建 asyncio 任务跑 `PMAC.xxx` → 结果写日志。
- **上行（状态）**：`_poll_loop`（`gui_motors.py:892`）每 `POLL_S=0.5s` 调 `motor_status_many(ALL_MOTORS)` → 分发到各 `MotorControl.update_status`、`OverviewTab.update_status`，并对 `MainWindow._viz_tabs`（带 3D 的装置页列表）里每页调 `push_positions(st)`，由该页按自己的电机集合更新 3D。

### 6.4 视觉主题

全局 QSS 字符串 `THEME`（`gui_motors.py:78`）在 `main()` 里 `app.setStyleSheet(THEME)` 注入，统一了配色、圆角、留白、按钮层级（普通 / `kind=primary` / `kind=danger`）、表格、滚动条等。值芯片与状态色用独立常量 `_VAL_STYLE / _OK_BG / _BAD_BG`（`gui_motors.py:66`）。数字统一经 `fnum / fsign`（`gui_motors.py:129`）格式化：最多 5 位小数、去尾随 0、大数不走科学计数。

## 7. 3D 可视化

三套 3D 视图同构、互相独立，按装置 `key` 取资产（`viewer_assets(key)`）：升降台（`viewer/`，电机 5）、机械快门（`viewer/shutter/`，电机 6）、调焦（`viewer/focus/`，电机 1-4）。各自一份 `Viewer.qml` + `baked/` + `model.glb`。下面 §7.1-7.4 以单电机的升降台为例；调焦的多电机/堆叠特例见 §7.5。

### 7.1 模型预处理（balsam）

Qt 6.11 的 `RuntimeLoader` 运行时加载 GLB 时**丢失节点名**（`objectName` 为空），无法按名定位升降台子件。改用 PySide6 自带的 `balsam` 工具离线预烤：

```
model.glb  ──balsam──►  baked/Model.qml + baked/meshes/*.mesh
```

烤出的 `Model.qml` 保留全部中文 `objectName`，可在场景里精确找到"升降台转接台面 / 背板 / 重物支撑板"这 3 个运动件。

### 7.2 场景（`viewer/Viewer.qml`）

- `Component.onCompleted` 用 `Qt.createComponent("baked/Model.qml")` 动态加载场景挂到旋转节点 `modelOrient` 下。
- `processModel()` 对节点树做 BFS：
  - 命中 `hidePatterns`（螺丝 / 垫片 / 光源等）→ 隐藏；
  - 命中 `movingPatterns` → 记录初始位置，作为运动节点。升降台取 3 个升降件；快门只取"快门本体单元"父节点一支（平移它即带动整支挡光组件，BFS 排除其 `_Part` 子件以免重复平移）；
  - 收集全部材质，统一改成 `metalness=0` 的浅灰哑光——烤出来的 PBR 材质很多 `metalness=1`，没有环境贴图会渲成纯黑，这一步是模型可见的关键。
- 相机用 pivot + `MouseArea` 自己驱动 `camYaw / camPitch / camDist`（左键拖转视角、滚轮缩放）。不用 `OrbitCameraController`，因为它在 `QQuickWidget` 里收不到鼠标事件。

### 7.3 位移映射

```
deltaM (米) = (physPos - centerPhys) × mmPerUnit / 1000 × direction
```

`applyOffset()` 把 `deltaM` 沿 `axisName` 加到各运动节点的初始位置上。`axisName` 指模型坐标系的轴，各 `Viewer.qml` 自己决定它映射到屏幕的哪个方向：升降台用 `modelOrient(0,0,90)` 把模型 x 转到屏幕上下；快门不旋转、模型 x 就是屏幕左右（实测确认）。`mmPerUnit` 是纯视觉夸张倍率（快电机计数大，默认 0.0001 mm/count），不影响任何电机命令。

### 7.4 Python ↔ QML 桥（`viewer3d.py`）

`Viewer3DWidget(QQuickWidget)` 对单电机 viewer 暴露 `set_motor_position / set_center / set_scale / set_axis`；对多电机的调焦另暴露 `set_motor_pos(motor, phys) / set_motor_center(motor, center) / set_focus_dirs(dir_fb, dir_lr)`（写 QML 的 `posM{n} / centerM{n} / dirFB / dirLR`）。全部转成对 QML root 的 `setProperty`，由 QML 的 `onXxxChanged` 触发重算，**不走 `runJavaScript`**。

> **竞态处理**：`QQuickWidget.setSource` 是同步的，QML 可能在构造函数里就 Ready，此时外部还没来得及连 `ready_changed` 信号。因此 `_set_prop` 在未 Ready 时把调用缓存进 `_pending_props`，Ready 时统一回放（`viewer3d.py:94`）；`DeviceTab._build_viewer_pane` 也在创建后立即检查 `is_ready` 补一次配置下发。

### 7.5 调焦特例（`viewer/focus/Viewer.qml`，多电机 + 堆叠联动）

调焦页一套两份机构：电机 1/2 与 3/4 各一套。`Component.onCompleted` 把同一个 `baked/Model.qml` `createObject` 两次，分别挂到 `inst1 / inst2` 两个父节点（`inst2` 沿 x 偏移并排）。每套各自 BFS 分类节点：

收集每套的全部可见件，给每件打一个 `fb` 标记（是否也随前后动），运动规则（用户确认的物理）：

- **左右（Y，电机 2/4）**：整套**所有件一起**沿模型 x 平移（含最底下的底板）——堆叠台最底层的左右台带动其上全部，没有可见的固定件。
- **前后（X，电机 1/3）**：除**底板**（`二维电动平移台转接平板`，`fb=false`）外的所有件（相机 + 调焦台）沿模型 z 平移；底板不随前后动。

即每套 `applyOffset`：`dLR = (posM[ml]-centerM[ml])·scale·dirLR`、`dFB = (posM[mf]-centerM[mf])·scale·dirFB`；每件加 `dLR`，`fb` 件再加 `dFB`。

> **模型限制**：该 GLB 的多轴调焦台（`联谊五维调整台`）是单块刚性 mesh，前后台与左右台没分开建模，所以前后时整块调焦台都跟着相机动（理想只动其上层）——这是模型粒度所限；但底板正确地只随左右、不随前后。轴向（z=前后、x=左右）由探针实测，方向可在界面翻转。
>
> 演进记录：v1 让「调焦台只左右、相机只前后」→ 前后时相机悬空；v2 让「调焦台+相机一起前后+左右、底板固定」→ 前后 OK 但左右时底板没跟着动；v3（当前）按用户确认改为「左右=全动含底板、前后=除底板外都动」。

## 8. 并发模型

- **qasync** 把 Qt 事件循环与 asyncio 事件循环合二为一（`main()`，`gui_motors.py:928`）。所有按钮回调用 `@asyncSlot` 修饰，可直接 `await`。
- **串行命令**：`PMAC._lock` 保证 gpascii 会话上一次只跑一条命令。
- **在飞任务**：`MainWindow._inflight` 记录当前命令任务；`_set_busy(True)` 在命令执行期间禁用动作按钮（去使能/急停除外）。
- **软急停**：`soft_estop(motor)`（`gui_motors.py:767`）先 `cancel()` 在飞任务，再发 `#nk`。一键去使能（`_on_disable_all`）同理：先取消在飞，再逐台去使能。
- **轮询**：`_poll_loop` 是独立 asyncio 任务，连接成功后启动，断开时取消。单次轮询失败只记日志、睡 `POLL_S` 后继续，不中断循环。

## 9. 日志

日志不在前端显示（早期文本框在高分屏下鼠标选中会偏行）。`MainWindow.log`（`gui_motors.py:688`）做三件事：

1. `print(line, flush=True)` 打到启动终端（可直接复制）；
2. 追加到内存缓冲（5000 行上限）；
3. 若已勾选"保存日志"，立即写文件并 flush。

勾选保存（`_on_toggle_savelog`，`gui_motors.py:701`）时新建带时间戳的 `logs/pmac_YYYYMMDD_HHMMSS.log`，先把本次会话已有缓冲写入，之后实时追加。每次勾选生成**新文件**，不覆盖历史。

`snap.py` / `snap_gui.py` 是离屏渲染截图工具：在真实显示会话上加载界面、注入假状态、`grab()` 存 PNG。无显示的 SSH 环境下用于自测布局、3D 渲染与升降映射，无需人工肉眼检查。

## 11. 关键设计决策汇总

| 决策 | 原因 |
| --- | --- |
| 绝对移动用小写 `#nj=` | v1.2 协议与硬件一致；早期改大写是误判 |
| 绝对输入用"相对中心"语义、锁死 ±range | 固件 0 点在物理 offset 处，直接填物理值易冲限位 |
| 范围内不弹确认、只拦超界 | 频繁操作时确认弹窗很烦 |
| 全 8 台批量读状态 | 单次往返代替 40 次，轮询才跟得上 |
| 3D 用 balsam 预烤而非 RuntimeLoader | 运行时加载丢节点名，无法定位子件 |
| 材质统一改 `metalness=0` | 烤出的金属材质无环境贴图会渲成黑 |
| 相机用 MouseArea 自驱 | `OrbitCameraController` 在 `QQuickWidget` 里不收事件 |
| 日志移出前端、打终端 + 可存文件 | 高分屏文本框选中偏行；终端 + 文件够用 |
| `motor_config.json` 不进 git | 运行态本机覆盖；默认值已在代码里 |

## 12. 已知限制

- 仅支持单台控制器、单连接、串行命令。
- 电机 5–8 的 offset / 限位暂为占位（0 / ±2,000,000 安全帽），待硬件标定后在界面里改。
- 3D 模型目前有三套："光栅切换装置"（升降台）、"机械快门装置"（挡光片）、"调焦装置"（两套调焦机构）；哈特曼门页暂无 3D。再加一套：在 `viewer/` 下放新目录、`VIZ_DEVICES`/`DEFAULT_VIZ_BY_DEVICE` 按装置 key 加一项即可，界面侧无需改。
- 调焦 GLB 的多轴台是单块刚性 mesh，3D 只能还原「相机前后 + 整台左右」的运动语义，无法显示前后台相对左右台滑动（见 §7.5）。
- `requirements.txt` 里关于 QtWebEngine 的注释是历史遗留——当前用的是 QtQuick3D，不依赖 QtWebEngine（见 §13）。
