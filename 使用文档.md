# PMAC 多装置控制软件 — 使用文档

| 项 | 内容 |
| --- | --- |
| 适用代码 | `test3/` |
| 运行入口 | `python gui_motors.py` |
| 控制器 | `192.168.0.200`（SSH root / deltatau，软件内已写死） |

本文面向操作人员，讲清楚怎么装、怎么连、怎么动电机、各页面怎么用。涉及电机运动的部分请在确认机械周边安全后操作。

---

## 1. 简介

软件连接 CK3M 控制器，控制 8 个电机。8 个电机按机械装置分 4 组，每组一个标签页：

| 标签页 | 电机 |
| --- | --- |
| 调焦装置 | 1、2、3、4（带 3D：两套调焦机构，前后/左右联动） |
| 光栅切换装置 | 5（带 3D 升降预览） |
| 机械快门装置 | 6（带 3D 挡光预览） |
| 哈特曼门装置 | 7（左）、8（右）（带 3D：两扇门绕轴旋转） |

外加两页：「总览」——一键使能/去使能所有电机 + 全部 8 台实时状态表；「整机」——
整个光谱仪的 3D 模型，8 个电机的运动全部联动显示，并可叠加光路演示（入射白光经
快门、哈特曼门、准直镜筒，分色镜分成蓝/红两支，各自经光栅/棱栅与镜筒进入两台相机）。

> **截图待补充 — 整机页（半透明外壳 + 光路 + 各装置联动）**

整机页说明：
- 外壳自动半透明，能看见内部装置与光路；底部复选框可开关光路显示。
- 各装置的显示比例/方向**沿用各装置页的设置**，装置页里调了这里立即跟着变。
- 视角操作与其它页相同（左键转/右键平移/滚轮缩放，自动记忆）。
- 模型约 1600 个零件，**首次进入加载约 5 秒**属正常。

## 2. 运行环境

- 操作系统：Linux 桌面（在 Wayland 或 X11 图形会话下运行；3D 预览需要真实显卡）。
- Python 3.10+。
- 依赖：`asyncssh`、`PySide6`（含 QtQuick3D / QtQuickWidgets）、`qasync`。
- 程序所在路径不要含中文（Qt 子进程对非 ASCII 路径有兼容问题）。

---

## 3. 安装

```bash
# 在程序目录下
python -m venv venv
source venv/bin/activate
pip install asyncssh PySide6 qasync
```

> `requirements.txt` 里有一条关于 QtWebEngine 的旧注释，可忽略——当前版本用的是 QtQuick3D，随 PySide6 一起装好，不需要额外操作。

## 6. 标准操作流程

1. **连接**：点顶栏「连接」。成功后圆点变绿、状态变"已连接 192.168.0.200"，开始每 0.5 秒刷新所有电机状态。
2. **启动 EtherCAT**：点「启动 EtherCAT」（发送 `ECAT[0].enable`）。
3. **使能电机**：
   - 在「总览」页点「一键使能所有电机」，或
   - 进到对应装置页，点某个电机的「使能」。
   - 电机 1–4 不依赖 EtherCAT 使能即可动，但电机级使能/去使能按钮对所有电机都有效。
4. **移动**：在电机卡片里填目标值，点「绝对移动」或「相对移动」。
5. **结束**：需要急停时点电机的「去使能（软急停）」或总览页「一键去使能」；用完点顶栏「断开」。

## 7. 总览页

> **截图待补充 — 总览页（含一键使能/去使能按钮与 8 行状态表）**

- **一键使能所有电机**：依次给 #1–#8 发使能。
- **一键去使能所有电机（软急停）**：先取消正在执行的命令，再依次给 #1–#8 去使能。该按钮在执行其它命令期间仍可点。
- **状态表**：每行一个电机，列为 `电机 / 名称 / ActPos / ActVel / AmpEna / AmpFault / SoftLimit`。
  - `AmpEna`：1=已使能（绿），0=未使能（红）。
  - `AmpFault` / `SoftLimit`：0=正常（绿），非 0=异常（红）。

---

## 8. 电机控制卡片

每个装置页里，一个电机是一张卡片，分三区。

> **截图待补充 — 单个电机控制卡片（状态 / 运动 / 参数 三区）**

### 状态区

实时显示 5 个量：

| 字段 | 含义 |
| --- | --- |
| ActPos | 编码器实测位置 |
| 相对中心 | `ActPos - offset` 的换算值，单独一行显示 |
| ActVel | 实测速度 |
| AmpEna | 使能状态（绿=1 / 红=0） |
| AmpFault | 故障（绿=0 正常） |
| SoftLimit | 限位触发（绿=0 正常） |

### 运动区

- **使能 / 去使能**：`#nj/` / `#nk`。去使能即软急停。
- **绝对移动**：填"相对中心"的目标值（见 §9），点按钮发 `#nj=`。
- **相对移动**：填增量，点按钮发 `#nj:`。

### 参数区

- **offset**、**范围±**：可改、点「保存」即时生效并写入 `motor_config.json`，下次启动自动加载。电机工程师改了标定值，在这里改即可，不必动代码。

---

## 9. 坐标与范围（重要）

绝对移动框里填的是**相对中心**的值，不是物理读数：

```
你填的值 X（限制在 -范围 ~ +范围）
实际物理位置 ≈ X + offset
```

举例（电机 1，offset=17，范围 ±6）：

| 填入 | 物理落点 |
| --- | --- |
| 0 | 17 |
| +6 | 23 |
| -6 | 11 |

电机 5–8 的 offset=0，填的值就是物理值，范围 ±2,000,000。

- 绝对移动：超出 `±范围` 会被拦下并弹提示，不下发。
- 相对移动：`当前物理位置 + 增量` 超出 `[offset-范围, offset+范围]` 同样被拦。
- 范围内的移动直接执行，不再二次确认。

## 10. 3D 预览（全部 4 个装置页）

> **截图待补充 — 调焦装置页（左侧两套 3D 调焦机构 + 右侧电机 1-4 卡片）**

> **截图待补充 — 光栅切换装置页（左侧 3D 升降模型 + 右侧电机 5 卡片）**

> **截图待补充 — 机械快门装置页（左侧 3D 快门模型 + 右侧电机 6 卡片）**

> **截图待补充 — 哈特曼门装置页（左侧 3D 两扇门 + 右侧电机 7/8 卡片）**

每个装置页左侧都有 3D 模型，随对应电机的 `ActPos` 实时联动，只动核心运动件、机架不动：

- **调焦装置（电机 1-4）**：两套机构并排，每套调焦台分上下两层。**左右**（y 电机 2/4）时整套所有件一起动（下层带着上层和相机一起走）；**前后**（x 电机 1/3）时只动相机 + 上层调焦台，下层调焦台和底板不动。
- **光栅切换装置（电机 5）**：升降台台面随电机上下移动。
- **机械快门装置（电机 6）**：挡光片随电机左右滑动，盖住 / 让开中间的光孔。
- **哈特曼门装置（电机 7/8）**：两扇黑色门板**绕各自门轴旋转**（唯一的旋转装置），电机 7=左门、电机 8=右门，互相独立。

操作通用：

- **鼠标**：左键拖=转视角，**右键拖=平移模型位置**，滚轮=缩放。**视角会自动记住**——调完自动存盘（含平移位置），下次启动直接恢复你上次调好的角度、大小和位置，不用每次重调。
- **显示比例**：放大倍率，纯视觉、不影响电机命令。看不清就调大（调焦默认 8 mm/count；光栅/快门是快电机、计数大，默认 0.0001 mm/count；哈特曼门是旋转，单位为 **°/count**，默认 0.0001）。
- **方向**：光栅/快门为单方向「方向：±（点击翻转）」；调焦为「前后方向 / 左右方向」、哈特曼门为「左门方向 / 右门方向」，各自独立翻转，装反了点对应那个。

> 调焦的 3D 是「大体框架」示意：原模型里多轴调焦台本是一整块，为还原前后/左右两级运动，已离线把它劈成上下两层。上下层的分界高度是估算的，看着不准可以让我调。

每页的这组显示参数各自独立保存，与 offset/范围一样写进 `motor_config.json`，下次启动恢复。

> 3D 运动做了平滑插值（数据 0.5s 刷新一次，画面在间隙里连续过渡），所以模型动画比真实位置约滞后半秒——这只影响预览观感，数值区显示的始终是真实读数。

## 11. 日志

- 启动终端**始终**实时打印日志，可直接选中复制。
- 勾选底部「保存日志到文件」后，本次会话日志写入 `logs/pmac_YYYYMMDD_HHMMSS.log`（含勾选前已产生的日志）。
- 每次勾选生成新文件，**不覆盖**以前的日志。取消勾选即停止写入、关闭文件。

## 12. 配置文件

`motor_config.json`（程序目录下，运行时自动生成/更新）：

```json
{
  "motors": {
    "1": {"center": 17, "range": 6},
    "5": {"center": 0,  "range": 2000000}
  },
  "viz": {
    "5": {"mm_per_unit": 0.0001, "axis": "x", "direction": 1},
    "6": {"mm_per_unit": 0.0001, "axis": "x", "direction": 1}
  }
}
```

- 删掉该文件不影响使用，软件会用代码内置的默认值重建。
- 想改某电机的 offset/范围，优先在界面「参数区」改并保存，而不是手改文件。

## 13. 命令行工具（可选）

不开界面、单电机快速自检：

```bash
python cli_motor1.py status        # 只读 motor1 状态
python cli_motor1.py enable        # #1j/
python cli_motor1.py move-abs 0    # #1j=0
python cli_motor1.py move-rel 0.5  # #1j:0.5
python cli_motor1.py disable       # #1k
```

排查 SSH + gpascii 链路（连不上时用）：

```bash
python diagnose.py            # 逐步检查 TCP → SSH → shell → gpascii
python diagnose.py --verbose  # 打开 asyncssh 详细日志
```

## 14. 常见问题

| 现象 | 排查方向 |
| --- | --- |
| 点「连接」失败 | 先 `ping 192.168.0.200`；再 `python diagnose.py` 看卡在哪一步 |
| 连上但状态全是 `--` | 等首次轮询（≤0.5s）；若一直如此看终端日志有无 `poll 失败` |
| 电机 5–8 移动报错 | 这几台必须先「使能」，未使能下发会被固件拒绝 |
| 绝对移动点了没反应 | 多半是超范围被拦，看弹窗与终端日志；确认填的是"相对中心"值 |
| 3D 区域空白 / 报错 | 需在图形会话下运行；纯离屏不渲染。看终端有无 QML 报错 |
| 3D 升降方向反了 / 看不清 | 调「显示比例」、切「运动轴」、点「方向」翻转 |
| 启动报路径相关错误 | 确认程序目录路径不含中文 |
